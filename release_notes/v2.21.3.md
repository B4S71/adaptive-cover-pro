# v2.21.3

## 🔑 Highlights

- Venetian covers now support explicit tilt on custom position slots and on the default/sunset path — solar-computed tilt is still the default, but you can pin slats to any angle for specific scenes such as movie mode or cleaning mode.
- The proxy cover entity suffix changes from `_slider` to `_managed`, reflecting that the proxy handles open, close, and tilt commands — not just slider moves. Existing entity IDs survive unless you manually rename them.
- 3221 tests passing.

## ✨ Features

- **Explicit tilt for venetian custom positions and default/sunset path (#369, #381):** Custom position slots 1–4 now accept an optional tilt value (`custom_position_tilt_1` … `custom_position_tilt_4`, 0–100 %, `None` = use solar tilt). When a custom position handler fires and a tilt value is configured, `VenetianPolicy.post_pipeline_resolve` honors the handler-supplied tilt directly — skipping the solar-tilt calculation — and emits a `venetian_handler_tilt` trace step in the Decision Trace. The same mechanism applies to the default and sunset paths via two new global options: `default_tilt` (applied whenever no handler claims the cycle) and `sunset_tilt` (applied during the sunset window, falling back to `default_tilt` if not set). All six new options are optional; omitting them preserves the existing solar-computed tilt behavior. This unlocks pinned-angle scenes — for example, a "movie mode" binary sensor that drives position 0 %, tilt 0 % — without needing a separate manual override.

## 🔧 Refactor

- **Proxy cover entity suffix renamed: `_slider` → `_managed` (#374, #382):** The opt-in proxy cover entity display name changes from `"<title> Slider"` to `"<title> Managed"`, and the internal trigger identifiers are extracted from inline strings into named constants (`TRIGGER_PROXY_POSITION`, `TRIGGER_PROXY_OPEN`, `TRIGGER_PROXY_CLOSE`, `TRIGGER_PROXY_TILT` in `const.py`). The rename better represents what the proxy actually does: it intercepts open, close, set-position, and set-tilt-position commands, not just slider moves. The proxy's `unique_id` is unchanged, so the underlying registry row survives the update and the display name refreshes automatically. **Heads-up:** if you added proxy cover entities by ID to automations, scripts, or dashboards before this release, your entity IDs remain `cover.<name>_slider` until you rename them manually in **Settings → Entities**. The new ID format is `cover.<name>_managed`. Translation descriptions for the `enable_proxy_cover` toggle updated across EN/DE/FR.

## 🧪 Testing
