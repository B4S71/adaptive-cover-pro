# v2.21.4

## 🔑 Highlights

Two venetian regressions from v2.21.3 are resolved. KNX, Shelly 2PM, and FGR223 users who saw spurious manual overrides during blind travel — or covers sitting at the wrong slat angle after the tilt-first opening sequence — should update immediately. A separate fix closes a time-window bypass that let custom-position sensor edges move any cover type before the configured start time.

## 🐛 Bug Fixes

- **Venetian suppression cap fires against in-transit motor drift (#33, PR #386):** `is_in_suppression_with_cap` evaluated the 30% back-rotate delta cap even while the cover's HA state was `opening` or `closing`. Real-world venetian actuators (KNX, Shelly 2PM, FGR223) back-retract slats by well over 30% during carriage travel, so the cap fired against pure motor drift and the manual-override path latched onto it — producing spurious override events while the blind was still moving. The fix bypasses the cap whenever `cover.state` is in the moving set (`_COVER_MOVING_STATES`); the cap reasserts once the carriage settles, so genuine user moves post-settle still trip the override as intended.

- **Verify step silently skipped on tilt-first opening path (#33, PR #386):** `_send_tilt_command`'s target-unchanged dedup returned early without running the post-settle verify step when the same tilt target had been sent as a fire-and-forget by `before_position_command`. If the actuator landed at the wrong tilt angle, drift was never detected, the stored target was never cleared, and the cover sat at the wrong angle indefinitely while also compounding with the cap bug above. The fix introduces `_tilt_targets_verified` — a set of entity IDs whose stored target has been confirmed against the actuator. The dedup branch now runs verify when the entity is absent from that set, firing it exactly once per target. The published 2-service-call-per-cycle invariant for opening transitions is preserved.

- **Custom-position sensor edges bypass start/end time window (#383, PR #385):** `custom_position_sensor_triggered` and `custom_position_released` were listed in the time-window bypass disjunction in `coordinator.py`, letting sensor edges move covers outside the configured start/end window. At sunrise the calculated default flips from `sunset_pos` to the daytime default, so a release edge arriving before `start_time` dispatched the daytime position immediately — opening covers before the user's intended window. Both flags are removed from the bypass list but remain in the `use_force` block, so commands that arrive inside the window still bypass the time/delta gates as intended.

## 🧪 Testing

- ✅ 3229 tests passing.
- New tests in `tests/test_managers/test_dual_axis_sequencer.py` cover in-transit cap suppression (open and close states, large deltas), post-settle override re-engagement, and the dedup-with-verify path (`test_dedup_after_unverified_send_still_runs_verify`, `test_run_sequence_after_tilt_first_runs_verify_on_dedup`).
- New tests in `tests/test_manual_override_venetian.py` cover the combined cap + verify interaction for misbehaving actuators.
- New tests in `tests/test_coordinator_integration.py` cover release and trigger edges before `start_time`, after `end_time`, and during an active sunset window (`test_release_outside_window_pre_start_no_command`, `test_trigger_outside_window_pre_start_no_command`, `test_release_outside_window_sunset_active_no_command`).
