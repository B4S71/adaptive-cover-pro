# v2.22.0-beta.4

## 🎯 Should I install this?

This is a targeted follow-up to beta.3. It lands two bug fixes — one for MODE2 tilt covers with climate mode enabled, one closing the last false-override vector on venetian covers — plus a logo chore.

- **If you run a tilt cover (MODE2) with climate mode and a `min_position` set**, install this. The climate handler was computing slat positions that ignored which side of the window the sun was on, and a `min_position` ≥ 50 would collapse every climate decision to the open-slat position ([#373]).
- **If you run a venetian cover and saw spurious manual-override events during automation runs**, install this. Motor drift and verification-poll republishes inside the 5 s command-grace window were still able to trip the tilt axis, even after beta.2 fixed the position axis ([#33]).
- **If you are on beta.3 and neither case applies**, the update is low-risk — no user-visible behaviour changes for other cover types.
- **If you are on v2.21.5 without any active issue**, stay there. This beta validates the full refactor before a stable release.

## 🔑 What's new in beta.4

### 🐛 Bug Fixes

- **MODE2 tilt climate handler: sun-side awareness + `min_position` warning ([#373], PR [#405]):** The climate pipeline's `tilt_with_presence` and `tilt_without_presence` methods computed tilt percentages using formulas that were MODE1-only and that did not consider the sign of the sun's azimuth offset (gamma). In MODE2, 50% is the open (horizontal) slat position and both poles are closed, so choosing the wrong hemisphere produces a position on the open side rather than the blocking side. The summer-with-presence branch returned a static 75% regardless of which hemisphere the sun was in. The fix adds `TiltPolicy.climate_tilt_percentage` as a single conversion helper that picks the correct MODE2 hemisphere from the sign of gamma and routes all four call sites in the climate handler through it. A separate `min_position` footgun: in MODE2 any `min_position` ≥ 50 clamps every below-centre result back to the open slat, making climate and glare control silently ineffective. The config-flow summary now surfaces a ⚠️ warning when MODE2 is active and `min_position` ≥ 50.

- **Venetian tilt axis: suppress override during command grace ([#33], PR [#406]):** After beta.2 added cap-grace suppression to the position axis, the tilt axis still lacked a command-grace gate. Motor drift and verification-poll republishes inside the 5 s window were reaching `handle_state_change` without being filtered, and the tilt delta was large enough to register as a user touch. `VenetianPolicy.secondary_axis_check` now builds a suppression callback that OR-composes the existing back-rotate window (`is_in_tilt_suppression`) with a new `_is_in_tilt_command_grace` helper that delegates to `GracePeriodManager.is_in_command_grace_period`. `handle_state_change` accepts the `is_in_command_grace` predicate from the coordinator and emits `manual_override_rejected_command_grace` when it fires.

### 🧹 Chores

- **"Pro" suffix added to `dark_logo.png`:** Updated the dark-mode logo asset to include the "Pro" product name.

## 📖 Cumulative beta scope

beta.4 is cumulative. Everything in [v2.22.0-beta.1], [v2.22.0-beta.2], and [v2.22.0-beta.3] is included:

- **Nine-phase internal refactor (Phases A–H, beta.1/beta.2):** Coordinator responsibility distributed into `managers/`, `state/`, `pipeline/`, and `cover_types/`. Coordinator is ~640 lines lighter vs v2.21.5. Manual-override detection in `managers/cover_command/state_classifier.py`; cover-type-specific behaviour on `CoverTypePolicy` subclasses; cover-type literal branching eliminated. Net diff for the refactor: 62 files, +3,682 / −1,251 lines.

- **Post-settle cap grace — false venetian override fix ([#33], beta.2):** `DualAxisSequencer.is_in_suppression_with_cap` holds the cap bypass active for 5 seconds after a position command stamp, preventing KNX, Shelly 2PM, FGR223, and similar actuators from tripping false manual-override when `cover.state` settles before the tilt-walk burst finishes publishing.

- **Phase I — config-flow plumbing refactor (beta.3):** Collapsed duplicated schema reads, step routing, and options-step boilerplate into shared helpers, reducing `config_flow.py` by 168 lines.

- **Legacy climate step + optional-key cleanup (beta.3):** Removed `async_step_climate` and `CLIMATE_SCHEMA` dead code; extracted optional-key constants for `WEATHER_OVERRIDE`, `LIGHT_CLOUD`, and `TEMPERATURE_CLIMATE` schemas to prevent [#377]-class drift; fixed a latent omission where `CONF_CLOUDY_POSITION` was missing from its inline optional-key list.

- **"Is Sunny" sensor can now be cleared in the options flow ([#377], beta.3):** `CONF_BINARY_SENSOR` added to the optional-entities list so a blank submission removes the prior value instead of silently keeping it.

- **MODE2 tilt climate handler fixes and `min_position` warning (beta.4, above).**

- **Venetian tilt-axis command-grace suppression (beta.4, above).**

## 🧪 Testing

3397 tests passing (up from 3384 in v2.22.0-beta.3).

New in beta.4:

- `test_false_manual_override_on_automation.py`: two new tests — position change inside command-grace window does not trip override; position change after grace expired does trip override (the negative case).
- `tests/test_manual_override_venetian.py`: four regression tests covering `tilt_only` and `position_and_tilt` modes, primary-axis position-during-grace, and the post-grace negative case.
- `tests/test_cover_types/test_venetian.py`: updated `test_secondary_axis_check_carries_expected_tilt` to assert the suppression callback is an OR-composition and returns `False` when neither the sequencer nor a grace manager is attached.
